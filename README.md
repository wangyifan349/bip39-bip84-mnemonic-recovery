# BIP39 / BIP84 Bitcoin Mnemonic Recovery Tool

A simple, fully offline Bitcoin mnemonic recovery tool for recovering missing BIP39 mnemonic words when you know most of the words and at least one generated Bitcoin Native SegWit address.

This tool is designed for Bitcoin mainnet BIP84 wallets, which usually generate Native SegWit addresses starting with `bc1q`.

Default derivation path:

```text
m/84'/0'/0'/0/index
````

## What This Tool Does

This tool helps you recover a BIP39 mnemonic when a small number of words are missing.

You provide:

1. A known Bitcoin receiving address
2. A mnemonic word list
3. Question marks `?` for unknown word positions
4. An optional candidate word list
5. An optional BIP39 passphrase
6. An address index range to check

The script then tries possible word combinations, validates each candidate mnemonic using the BIP39 checksum, derives BIP84 Bitcoin addresses, and checks whether any derived address matches the target address.

## Fully Offline by Design

This program is designed to run completely offline.

It does not:

* Connect to the internet
* Call any remote API
* Query blockchain explorers
* Send mnemonic words anywhere
* Upload wallet data
* Download external data at runtime
* Write recovery results to local files

All computation is performed locally on your own machine.

The only external dependency is the Python package `bip-utils`, which must be installed before running the script. For real wallet recovery, it is strongly recommended to install all dependencies in advance and run the script on an offline computer.

## Suitable Use Case

This tool is useful when:

* You own the wallet.
* You know most of your BIP39 mnemonic words.
* A few word positions are missing or uncertain.
* Unknown positions can be marked with `?`.
* You know at least one Bitcoin receiving address generated by the wallet.
* The address is a Native SegWit Bitcoin address starting with `bc1q`.
* The wallet uses the BIP84 derivation path.

Example:

```python
["abandon", "abandon", "?", "abandon", "abandon", "abandon",
 "abandon", "abandon", "abandon", "abandon", "abandon", "?"]
```

In this example, the 3rd and 12th words are unknown.

## Important Security Warning

Use this tool only to recover your own wallet.

A mnemonic phrase is not just a password. It is effectively the final control key to the wallet assets. Anyone with the correct mnemonic, and the correct BIP39 passphrase if one was used, can control the funds.

If you are using this tool because you lost some mnemonic words, you may be under serious stress. Losing mnemonic words is painful, but it is also a serious reminder that mnemonic backup and storage must be handled carefully.

This kind of problem is usually caused by careless backup or storage practices, such as:

* Writing words unclearly
* Losing part of the backup
* Storing only one copy
* Saving the mnemonic in screenshots
* Saving it in cloud storage
* Saving it in chat history
* Saving it on an online device
* Confusing the word order
* Mixing up similar BIP39 words

If you successfully recover your wallet, treat it as a serious lesson. Back up the mnemonic offline, clearly, accurately, redundantly, and store it in a secure physical location.

## Features

* Fully offline execution
* No network access
* No API calls
* No blockchain explorer requests
* No automatic file output
* Supports BIP39 English mnemonics
* Supports 12, 15, 18, 21, and 24 word mnemonics
* Uses `?` for unknown word positions
* Supports a custom candidate word list
* Can try all 2048 English BIP39 words if no candidate list is provided
* Validates BIP39 checksum before deriving addresses

* Derives Bitcoin BIP84 Native SegWit addresses
* Searches multiple address indexes

## Requirements

Python 3.8 or newer is recommended.

Install the required dependency:

```bash
pip install bip-utils
```

## Offline Installation

If you want to run this tool on an offline computer, prepare the dependency packages on another machine first.

On an online machine:

```bash
mkdir bip_utils_offline

python -m pip download --only-binary=:all: --dest bip_utils_offline bip-utils
```

Then copy the `bip_utils_offline` folder to the offline machine.

On the offline machine:

```bash
python -m pip install --no-index --find-links bip_utils_offline bip-utils

```

```bash
python -c "import sys, platform, bip_utils; from bip_utils import Bip39MnemonicGenerator, Bip39WordsNum, Bip39SeedGenerator, Bip44Changes, Bip84, Bip84Coins; m=str(Bip39MnemonicGenerator().FromWordsNumber(Bip39WordsNum.WORDS_NUM_12)); seed=Bip39SeedGenerator(m).Generate(''); addr=Bip84.FromSeed(seed, Bip84Coins.BITCOIN).Purpose().Coin().Account(0).Change(Bip44Changes.CHAIN_EXT).AddressIndex(0).PublicKey().ToAddress(); print('Python executable:', sys.executable); print('Python version:', sys.version.replace('\n',' ')); print('Platform:', platform.platform()); print('bip_utils version:', getattr(bip_utils, '__version__', 'unknown')); print('Generated 12-word mnemonic:', m); print('BIP84 path:', \"m/84'/0'/0'/0/0\"); print('Generated BTC address:', addr); print('Result: bip-utils is installed and working.')"
```


After installation, the recovery script itself can run without internet access.

## Usage

Run the script:

```bash
python mnemonic_recovery.py
```

The script will ask for:

1. Target BTC address
2. Mnemonic word list
3. Candidate word list
4. Optional BIP39 passphrase
5. Start address index
6. Number of addresses to check

## Input Example

Target address:

```text
bc1qqxwmzs7qyatpht84hqmavkag0r3gnalyjxqr9d
```

Mnemonic word list:

```python
["abandon", "abandon", "?", "abandon", "abandon", "abandon",
 "abandon", "abandon", "abandon", "abandon", "abandon", "?"]
```

Candidate word list:

```python
["about", "above", "absent"]
```

If you do not know the missing word candidates, press Enter when asked for the candidate word list.

In that case, the script will try all 2048 English BIP39 words for each unknown position.

## Candidate Word List Behavior

The same candidate word list is applied to every `?` position.

For example, if the mnemonic has two unknown positions and the candidate list is:

```python
["about", "above"]
```

The script will try:

```text
about about
about above
above about
above above
```

This keeps the script simple and easy to use.

## Search Complexity

Be careful with the number of missing words.

If no candidate list is provided, the script uses all 2048 English BIP39 words.

Approximate search space:

```text
1 missing word  = 2,048 combinations
2 missing words = 4,194,304 combinations
3 missing words = 8,589,934,592 combinations
```

Three or more fully unknown words may take a very long time.

If you remember possible candidates for the missing words, provide a smaller candidate list.

Example:

```python
["about", "above", "absent", "access"]
```

With two unknown positions, this creates:

```text
4 × 4 = 16 combinations
```

This is much faster than trying the full BIP39 word list.

## BIP39 Checksum Validation

The script validates each candidate mnemonic using the BIP39 checksum before deriving addresses.

This is important because most random word combinations are not valid BIP39 mnemonics.

Invalid mnemonics are skipped immediately, which makes the search significantly faster.

## Derivation Path

This tool checks addresses derived from:

```text
m/84'/0'/0'/0/index
```

Meaning:

```text
84' = BIP84
0'  = Bitcoin mainnet
0'  = account 0
0   = external receiving address chain
```

By default, the script checks addresses such as:

```text
m/84'/0'/0'/0/0
m/84'/0'/0'/0/1
m/84'/0'/0'/0/2
...
```

You can choose the starting index and how many addresses to check when running the script.

## BIP39 Passphrase

The BIP39 passphrase is not the same as a wallet application password.

It is an optional extra phrase used during seed generation.

If you never set a BIP39 passphrase, leave it empty.

If you did set one, the correct mnemonic will not generate the correct wallet unless the correct passphrase is also provided.

## Limitations

This tool currently focuses on:

* Bitcoin mainnet
* BIP39 English mnemonic words
* BIP84 Native SegWit addresses
* `bc1q...` receiving addresses
* External receiving chain addresses
* Path format: `m/84'/0'/0'/0/index`

It does not currently support:

* Legacy Bitcoin addresses starting with `1`
* Nested SegWit addresses starting with `3`
* Taproot addresses starting with `bc1p`
* Ethereum addresses
* Electrum-specific mnemonic formats
* Non-English BIP39 word lists
* Per-position candidate word lists
* GPU acceleration
* Distributed search
