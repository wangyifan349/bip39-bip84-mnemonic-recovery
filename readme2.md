# BIP39 Mnemonic Recovery Toolkit

Offline Python tools for recovering BIP39 mnemonic phrases in several specific recovery cases.

The main script focuses on Bitcoin BIP39 / BIP84 Native SegWit wallet recovery when some mnemonic words are missing but their positions are known.

This repository also contains extra recovery scripts for related cases, such as mixed word order or 1-2 suspected wrong words. Those extra scripts are useful, but the primary use case of this repository is still offline BIP39 mnemonic recovery for wallets you own.

## Security Warning

Use this repository only for wallets you own or wallets that you are explicitly authorized to recover.

A mnemonic phrase is not a normal password. It is the master secret for a wallet. Anyone with the correct mnemonic phrase, and the correct BIP39 passphrase if one was used, may be able to control the funds.

For real recovery work:

* review the source code before entering a real mnemonic
* install the required dependency first
* disconnect from the internet before entering sensitive wallet data
* do not paste a real mnemonic into websites, chat apps, online notebooks, cloud tools, or AI tools
* do not run recovery on a VPS, shared computer, remote desktop, or machine you do not fully trust
* do not store recovered mnemonics in screenshots, cloud storage, browser notes, terminal logs, or chat history
* if the mnemonic may have been exposed, move the funds to a newly generated wallet after recovery

## What This Repository Does

The scripts work locally.

They generate candidate mnemonics, validate BIP39 checksums, derive addresses, and compare the derived address with a known target address.

They do not:

* connect to the internet during recovery
* call blockchain explorer APIs
* upload mnemonic words
* send wallet data anywhere
* download external data while recovering
* automatically write recovered mnemonics to files

The only required Python dependency is:

```bash id="5tb9sv"
bip-utils
```

## Main Recovery Case

Use `mnemonic_recovery.py` when:

* you know most of the BIP39 mnemonic words
* one or more word positions are unknown
* unknown positions can be marked with `?`
* you know at least one Bitcoin receiving address generated by the wallet
* the target address is a Bitcoin Native SegWit address, usually starting with `bc1q`
* the wallet uses the BIP84 external receiving path

The default derivation path is:

```text id="fmd707"
m/84'/0'/0'/0/index
```

The script supports checking an address index range.

For example, it can check:

```text id="6lselo"
m/84'/0'/0'/0/0
m/84'/0'/0'/0/1
m/84'/0'/0'/0/2
...
```

## Usage: Missing Mnemonic Words

Run:

```bash id="24x068"
python mnemonic_recovery.py
```

The script will ask for:

1. target BTC address
2. mnemonic template
3. candidate words for unknown positions
4. optional BIP39 passphrase
5. start address index
6. number of addresses to check

Example target address:

```text id="1d56sy"
bc1qqxwmzs7qyatpht84hqmavkag0r3gnalyjxqr9d
```

Example mnemonic template:

```text id="phvytq"
abandon abandon ? abandon abandon abandon abandon abandon abandon abandon abandon ?
```

You can separate words with spaces, English commas, or Chinese commas.

These formats are accepted:

```text id="xeah5x"
abandon abandon ? abandon
```

```text id="b1nhtt"
abandon, abandon, ?, abandon
```

```text id="x2d4vr"
abandon，abandon，？，abandon
```

Example candidate words:

```text id="xz08bf"
about above absent
```

The same candidate list is used for every `?` position.

If the candidate list is empty, the script tries all 2048 English BIP39 words for each unknown position.

## How the Main Script Works

`mnemonic_recovery.py` performs the recovery in this order:

1. reads the target Bitcoin address
2. reads the mnemonic template
3. checks that the mnemonic length is valid
4. checks that known words are valid English BIP39 words
5. finds all `?` positions
6. reads candidate words, or uses the full 2048-word English BIP39 list
7. generates every possible replacement combination
8. validates each completed mnemonic with the BIP39 checksum
9. skips invalid BIP39 mnemonics
10. derives Bitcoin BIP84 Native SegWit receiving addresses
11. compares each derived address with the target address
12. prints the recovered mnemonic if a match is found

The script does not simply compare word combinations blindly. It validates the BIP39 checksum first, which removes many invalid candidate phrases before address derivation.

## Search Complexity

The number of combinations grows quickly.

If no candidate list is provided:

```text id="fbm6mt"
1 missing word  = 2,048 combinations
2 missing words = 4,194,304 combinations
3 missing words = 8,589,934,592 combinations
```

This is before address derivation.

The BIP39 checksum removes many invalid candidates, but large searches can still take a long time.

If you remember possible words, use a short candidate list.

Example:

```text id="san82d"
about above absent access
```

With 2 unknown positions:

```text id="q0c7x2"
4 × 4 = 16 combinations
```

This is much faster than trying all 2048 words.

## BIP39 Passphrase

A BIP39 passphrase is not the same as a wallet app password.

If your wallet was created with a BIP39 passphrase, the same mnemonic words without that passphrase will derive a different wallet.

If you never set a BIP39 passphrase, leave the passphrase input empty.

If you did set one, you must enter it exactly. Spaces and capitalization can matter depending on what you originally used.

## Address Index Range

Many wallets generate a new receiving address every time you request one.

The first receiving address is usually:

```text id="nq2xcp"
m/84'/0'/0'/0/0
```

The next ones are:

```text id="hh1hg1"
m/84'/0'/0'/0/1
m/84'/0'/0'/0/2
m/84'/0'/0'/0/3
```

When running the main script, you can choose:

* start address index
* number of addresses to check

If your known address is not the first receiving address, increase the number of addresses to check.

## Other Recovery Cases

The repository includes additional scripts for related recovery situations.

### Mixed Word Order

Use `mnemonic_order_recovery.py` if:

* all words are known
* all words are spelled correctly
* the order is wrong or uncertain
* the target wallet is Bitcoin BIP84 Native SegWit
* the known address starts with `bc1q`

This script supports:

* 12-word and 24-word mnemonics
* fixed word positions
* adjacent word groups
* optional BIP39 passphrase
* address index range search

Example fixed position input:

```text id="cdsl94"
3=apple, 12=zoo
```

Example adjacent groups:

```text id="n2n4gl"
apple banana; zoo abandon about
```

An adjacent group means the words must stay together in that exact order, but the group position is still searched.

Run:

```bash id="xord4g"
python mnemonic_order_recovery.py
```

### Multi-Coin Order Recovery

Use `bip39_order_recovery.py` only if:

* all words are known
* every word is spelled correctly
* no word is missing
* no extra word exists
* only the order is wrong
* you want to check the first derived address

Run:

```bash id="v57el5"
python bip39_order_recovery.py
```

Supported modes:

```text id="0teclf"
BTC
ETH
ZEC
SOL
```

This script checks only address index `0`.

Supported address checks:

```text id="wfabfv"

BTC Native SegWit  bc1q...  m/84'/0'/0'/0/0
```

Use `lost2mnemonic.py` if:

## Choosing the Right Script

| Situation                                                                             | Script                       |
| ------------------------------------------------------------------------------------- | ---------------------------- |
| Some words are missing and their positions are known                                  | `mnemonic_recovery.py`       |
| All words are known but the order is wrong, BTC BIP84 `bc1q`                          | `mnemonic_order_recovery.py` |
| All words are known but the order is wrong, BTC / ETH / ZEC / SOL, first address only | `bip39_order_recovery.py`    |
| The phrase is complete but 1 or 2 words may be incorrect                              | `lost2mnemonic.py`           |

## Important Limitations

These tools do not guarantee recovery.

Recovery may fail if:

* the target address was not generated from the mnemonic
* the wrong derivation path is being checked
* the target address is outside the checked address index range
* the script only checks index `0` but your address is not index `0`
* the BIP39 passphrase is missing or incorrect
* too many words are missing, wrong, or unordered
* the real missing word is not in your candidate list
* the mnemonic uses a non-English BIP39 word list
* the wallet uses a non-standard derivation path
* the wallet is not a BIP39 wallet
* the address belongs to a different coin, account, or wallet

Some scripts check an address index range. Some scripts check only address index `0`. Read the script description before running a long search.

## Performance Notes

Missing-word recovery is usually more practical than full order recovery.

For missing words, the search space depends on:

```text id="simc8w"
candidate_count ^ missing_word_count
```

For order recovery, the search space may be factorial.

For 12 unique words:

```text id="b4d6wf"
12! = 479,001,600 possible orders
```

For 24 unique words, full brute force is normally not practical unless you have strong clues.

Fixed positions and adjacent groups can reduce the search space dramatically.
